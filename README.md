# Analys SaaS - Next.js Starter

## üìã Template Overview

**Analys SaaS** is a comprehensive, modern SaaS starter kit built with **Next.js 16**, **React 19**, and **Tailwind CSS 4**. It is designed to provide a robust foundation for building scalable web applications with a polished, premium aesthetic. This template includes pre-built pages, authentication integration, and a dynamic interface using Framer Motion.

## üìÑ Pages List

The template includes the following key pages:

- **Contact** (`/contact`): Contact form and information sections.
- **Pricing** (`/pricing`): Pricing tiers and feature comparison.
- **Authentication**:
  - Sign In (`/sign-in`)
  - Sign Up (`/sign-up`)

## üöÄ Installation

Follow these steps to get the project running locally:

1.  **Clone the repository**:

    ```bash
    git clone <repository-url>
    cd Analys-SaaS---next
    ```

2.  **Install dependencies**:

    ```bash
    npm install
    ```

3.  **Environment Setup**:
    Create a `.env.local` file in the root directory and add your required environment variables (e.g., Clerk keys):

    ```env
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
    CLERK_SECRET_KEY=sk_test_...
    ```

4.  **Run the development server**:

    ```bash
    npm run dev
    ```

5.  **Open the app**:
    Visit `http://localhost:3000` in your browser.

## üé® Customization Guide

- **Global Styles**: Modify `src/app/globals.css` to update global color variables, fonts, and base styles.
- **Layout**: The main layout wrapper is located in `src/app/layout.tsx`. Here you can modify the global Header, Footer, and theme providers.
- **Components**: Reusable UI components are found in `src/components/`.
- **Tailwind Config**: Since this project uses Tailwind CSS 4, configuration is handled via CSS variables and the `@theme` directive in your CSS files, or check `package.json` / legacy config if applicable.

## üîç SEO Setup

This template uses Next.js Metadata API for SEO optimization.

- **Default Metadata**: Configured in `src/app/layout.tsx`.
  ```typescript
  export const metadata: Metadata = {
    title: "Analyzer",
    description: "Generated by create next app",
  };
  ```
- **Page-Specific SEO**: You can export a `metadata` object from any `page.tsx` file to override defaults.
